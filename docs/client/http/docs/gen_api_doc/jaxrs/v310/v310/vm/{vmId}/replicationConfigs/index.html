Path: / v310 / vm / {vmId} / replicationConfigs

Path parameters:
vmId -
Method Summary
ResourceDescription
PUT /v310/vm/{vmId}/replicationConfigsUpdate the outgoing replication configurations for the VM specified by the given vmId to the new replication configurations, specified in the API input. This API will compare the old set of existing replication configurations and new set of replication configurations. Any replication configuration that exists in the new set, but is not found in the old set, will be added. Likewise, any replication configuration that is found in the old set, but is not found in the new set, will be removed. Replication configurations existing in both sets will be modified. This API does not allow setting isSystemDefault attribute of any replication configuration to true. If there is a need to re-enable system default replication on a VM, please use API. When configuring many outgoing replication configurations, DatastoreReplicationPath with a valid id is required for all replication configurations. Replication path ID of a DatastoreReplicationPath is used to uniquely identify a replication configuration in the system. Also, all replication configurations are required to be all enabled or disabled. Disable/enable of outgoing replication configuration of the VM is currently not supported. When configuring one outgoing replication configuration, DatastoreReplicationPath can be null only when the replication configuration is disabled. With this property, VM will not be eligible for system default replication, hence it will be completely unprotected.
