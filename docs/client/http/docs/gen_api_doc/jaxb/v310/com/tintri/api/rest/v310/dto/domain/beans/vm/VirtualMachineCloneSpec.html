Elements
NameTypeDescription
hypervHyperVCloneInfoHyper-V specific information if the clone operation involves Hyper-V VMs
remoteCopyInfoRemoteCopyInfoContains information needed for remote copy operation. Populate remoteCopyInfo object along with snapshotId.
restoreInfoRestoreInfoContains information needed for restore operation. Populated restoreInfo object along with snapshotId. Clone API with restoreInfo option is called on the destination of the clone. SnapshotId should be a UUID of replica. (replicated Snapshot)
rhevRhevCloneInfoRHEV specific information if the clone operation involves RHEV VMs
vmwareVMwareCloneInfoVMware specific information if the clone operation involves VMware VMs
Attributes
NameTypeDescription
consistencyCRASH_CONSISTENT | VM_CONSISTENT | APP_CONSISTENTThe consistency of the snapshot used by the clone API in case of clone from VM. SnapshotConsistency has 3 options. VM_CONSISTENT, CRASH_CONSISTENT or APP_CONSISTENT.
CRASH_CONSISTENTDisk activity quiesed snapshot
VM_CONSISTENTVM activity quiesed snapshot
APP_CONSISTENTApplication activity quiesed snapshot (currently not used)
countxsd:intCount represents number of cloned VMs to be created. Its value should be in the range of 1, 500. Mandatory for 'clone from VM' and 'clone from snapshot' to local datastore. The value will always be considered is 1 for 'remote copy' and 'mini DR'.
snapshotIdxsd:stringTintri Snapshot UUID. It is mandatory in case of 'clone from snapshot', 'remote copy (one shot copy)' and 'mini DR'
vmIdxsd:stringTintri VM UUID. It is mandatory for clone from VM. Optional otherwise.
